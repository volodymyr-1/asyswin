# AsysWin Enhanced v2

**Улучшенная система автоматизации рабочих процессов с AI-ассистентом и мониторингом провайдеров**

AsysWin Enhanced v2 записывает действия пользователя, анализирует их через AI провайдеров и генерирует Python-скрипты для автоматизации повторяющихся задач. Система поддерживает мониторинг провайдеров, автоматический fallback и оптимизацию настроек.

## Особенности

### 🤖 AI-ассистент
- Анимированный робот-ассистент с профессиональной графикой
- Звуковые эффекты для обратной связи
- Интеллектуальные рекомендации скриптов (топ-3)

### 📡 Мониторинг провайдеров
- Автоматическая проверка доступности провайдеров
- Реальное время мониторинга статуса подключений
- Автоматический fallback при ошибках
- Статистика использования и производительности

### 🎛️ Выбор моделей
- Поддержка различных моделей для каждого провайдера
- Автоматическая оптимизация выбора модели
- Сохранение предпочтений пользователя

### 🛡️ Безопасность
- Песочница для выполнения скриптов
- Проверка опасных операций
- Контроль ресурсов

### 📊 Аналитика
- Статистика использования
- Рекомендации по оптимизации
- История выполнения скриптов

## Установка

```bash
git clone <repository-url>
cd asyswin
pip install -r requirements.txt
```

## Настройка

### 1. AI провайдеры

#### Google Gemini (рекомендуется)
```bash
# Windows
set GEMINI_API_KEY=your_api_key_here

# Linux/Mac
export GEMINI_API_KEY=your_api_key_here
```
Получить ключ: https://makersuite.google.com/app/apikey

#### OpenAI GPT-4
```bash
# Windows
set OPENAI_API_KEY=your_api_key_here

# Linux/Mac
export OPENAI_API_KEY=your_api_key_here
```
Получить ключ: https://platform.openai.com/api-keys

#### Groq (бесплатный)
```bash
# Windows
set GROQ_API_KEY=your_api_key_here

# Linux/Mac
export GROQ_API_KEY=your_api_key_here
```
Получить ключ: https://console.groq.com/keys

#### LM Studio (локальный)
1. Установите LM Studio
2. Запустите сервер: Settings → Developer → Start Server
3. URL по умолчанию: `http://localhost:1234/v1`

### 2. Запуск

```bash
# Улучшенная версия с мониторингом провайдеров
python main_enhanced_v2.py
```

## Интерфейс

### Мониторинг провайдеров
- **Статус подключений**: В реальном времени отображается состояние каждого провайдера
- **Время ответа**: Показывает производительность каждого провайдера
- **Ошибки**: Информация о проблемах с подключением
- **Управление**: Тестирование и переключение провайдеров

### Настройки провайдеров
- **Выбор провайдера**: Переключение между доступными провайдерами
- **Выбор модели**: Выбор оптимальной модели для каждого провайдера
- **Тест подключения**: Проверка работоспособности провайдера
- **API ключи**: Безопасное хранение и управление ключами

### Оптимизация
- **Рекомендации**: Автоматические советы по улучшению производительности
- **Анализ настроек**: Проверка текущих параметров на оптимальность
- **Советы**: Рекомендации по выбору провайдеров и моделей

## Поддерживаемые провайдеры

### Google Gemini
- **Модели**: gemini-2.0-flash, gemini-2.0-pro, gemini-2.5-flash
- **Преимущества**: Высокое качество анализа, стабильность
- **Рекомендации**: Используйте gemini-2.0-flash для быстрых ответов

### OpenAI GPT-4
- **Модели**: gpt-4o-mini, gpt-4o, gpt-4-turbo
- **Преимущества**: Отличное качество, широкие возможности
- **Рекомендации**: Используйте gpt-4o-mini для экономии

### Groq
- **Модели**: llama3-8b-8192, llama3-70b-8192, llama-3.3-70b-versatile
- **Преимущества**: Бесплатный, очень быстрый
- **Рекомендации**: Используйте llama3-8b-8192 для баланса скорости и качества

### LM Studio
- **Модели**: auto, llama-3.2-3b, phi-4
- **Преимущества**: Локальное выполнение, нет ограничений
- **Рекомендации**: Используйте для конфиденциальных задач

## Разработка

### Добавление нового провайдера

1. Реализуйте интерфейс `AIProvider` в `llm_analyzer.py`
2. Добавьте модель в `provider_manager.py`
3. Обновите интерфейс в `settings_window_enhanced.py`
4. Добавьте тесты

### Добавление новой модели

1. Добавьте модель в `ModelInfo` в `provider_manager.py`
2. Обновите список моделей в настройках
3. Протестируйте совместимость
